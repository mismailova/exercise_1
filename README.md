# Конспект по Git <br>
*Уровень: для начинающих*


## Основные команды для работы с командной строкой

### Навигация	


- **pwd** - показывает текущую папку/директорию <br>
- **ls** - отображает содержимое директории (папки, файлы) <br>
- **ls -a** - показывает в том числе скрытые файлы <br>
- **cd first-project** - осуществляет переход в папку <br>
- **cd first-project/html** - для перехода в папку html <br>
- **cd ..** - переход на уровень выше (или с тильдой ~ - переход в домашнюю директорию) <br>
- **cd /** - переход в корневую директорию 


### Работа с файлами <br>
#### Создание
**touch index.html** - создает файл index.html (также возможно указать несколько файлов через пробел touch index.html README.md) <br>
**mkdir second-project** - создает папку second-project в текущей директории <br>
#### Копирование и перемещение
**cp file.txt ~/my-dir** - копирует файл в другое место <br>
**mv file.txt ~/my-dir** - перемещает файл в другое место <br>
#### Чтение <br>
**cat file.txt** - распечатать содержимое файла <br>
#### Удаление
**rm about.html** - удаляет файл <br>
**rmdir images** - удаляет директорию <br>
**rm -r second-project** - удаляет папку и все, что она содержит<br>


### Полезные возможности
- && - для разделения разных команд, которые можно перечислить списком 
- Стрелки вверх и вниз - буфер обмена команд в консоли 
- Tab - чтобы не вводить название папок полностью


## Работа с Git
### Инициализируем репозиторий git init <br>
Чтобы начать отслеживать изменения в проекте, папку с файлами *нужно сделать Git-репозиторием* <br>
**Шаги:** <br>
1. Перейти в нужную папку
2. Ввести команду **git init**
3. Зарегистрироваться на [GitHub](https://github.com)
4. Создать через интерфейс GitHub удаленный репозиторий
5. Выпустить SSH-ключ (от англ. *Secure Shell Protocol*) через терминал <br>

Команда, чтобы папка перестала являться репозиторием - **rm -rf .git** (удаление скрытой папки .git)<br>

Проверить состояние репозитория - **git status** (показывает текущее состояние репозитория) <br>

Подготовить файлы к сохранению - **git add README.md** <br>
Чтобы подготовить все файлы к сохранению - **git add --all** <br>
**git add .** - добавить всю текущую папку <br>

Выполнить коммит - **git commit** <br>
Коммит гарантирует, что изменения будут сохранены в истории и при необходимости к ним можно будет откатиться. Git add сообщает, какие файлы будут сохранены, а git commit их сохраняет <br>
**git commit -m 'текст сообщения'** - выполнить коммит с сообщением <br>

Посмотреть историю коммитов - **git log**

### Выпуск SSH через терминал и отправка изменений в удаленный репозиторий
_Предварительно проверить наличие ключей в домашней директории командой **ls -ls .ssh/**_ 
1. Перейти в домашнюю директорию
2. Сгенерировать ключи
```
$ ssh-keygen -t ed25519 -C "электронная почта, к которой привязан ваш аккаунт на GitHub"
```
3. Проверить создание **ls -a ~/.ssh** (файлом .pub можно делиться)
4. Скопировать содержимое файла с публичным ключом в буфер обмена
```
# скопировать содержимое ключа в буфер обмена:
$ pbcopy < ~/.ssh/id_rsa.pub
# для ed25519:
$ pbcopy < ~/.ssh/id_ed25519.pub
```
5. Перейти в настройки профиля в GitHub
6. Нажать слева на пункт *SSH and GPG keys*
7. Нажать  *SSH and GPG keys*
8. Заполнить *Title* (название ключа), *Key Type* (Authentication Key), *Key* (вставить публичный ключ из буфера)
9. Нажать *Add SSH key*
10. Проверить правильность ключа с помощью команды 
```
$ ssh -T git@github.com
```
11. Скопировать SSH url из GitHub (в репозитории) 
12. Связать локальный репозиторий и удаленный с помощью команды в терминале
```
$ cd ~/dev/first-project
$ git remote add origin git@github.com:%ИМЯ_АККАУНТА%/first-project.git
```
13. Убедиться, что репозитории связаны **git remote -v**
14. Отправить изменения на удаленный репозиторий - **git push** 
Для первого выполнения
```
$ git push -u origin main 
```
15. Проверить в GitHub в репозитории наличие измененных файлов и коммиты
